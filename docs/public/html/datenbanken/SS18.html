a)

Geben Sie fuer die Haltestelle Marktplatz die Anzahl der Gleise aus. Gehen Sie davon aus, dass in der Relation Properties dafuer genau ein Eintrag mit dem p_key "Anzahl Gleise" existiert.

xxxxxxxxxx
SELECT p_value
FROM Properties
WHERE halt_id = (SELECT id FROM Haltestelle WHERE name = 'Marktplatz')
AND p_key = 'Anzahl Gleise';

b)

Geben Sie die Namen aller Haltestellen ohne einen Eintrag in der Relation properties aus.

xxxxxxxxxx
SELECT name
FROM Haltestelle
WHERE id NOT IN (SELECT halt_id FROM Properties);

c)

Geben Sie name und p_key fuer alle Haltestellen aus, die zu einem p_key mehrere, nicht identische Werte (p_value) besitzen.

xxxxxxxxxx
SELECT Haltestelle.name, Properties.p_key
FROM Haltestelle
JOIN Properties ON Haltestelle.id = Properties.halt_id
GROUP BY Haltestelle.name, Properties.p_key
HAVING COUNT(DISTINCT Properties.p_value) > 1;

d)

Erstellen Sie die Sicht Gleise (Name, Anzahl_Gleise). Sie enthaelt die Namen aller Haltestellen mit dem entsprechenden Wert, identifiziert durch den p_key Anzahl Gleise. Haltestellen ohne solchen Eintrag sollen als (name, NULL) enthalten sein.

xxxxxxxxxx
CREATE VIEW Gleise AS
SELECT Haltestelle.name, Properties.p_value AS Anzahl_Gleise
FROM Haltestelle
LEFT JOIN Properties ON Haltestelle.id = Properties.halt_id AND Properties.p_key = 'Anzahl Gleise';

e)

Geben Sie die Namen der drei Haltestellen mit den meisten Einträgen (Tupel) in der properties-Relation aus.

xxxxxxxxxx
SELECT Haltestelle.name
FROM Haltestelle
JOIN Properties ON Haltestelle.id = Properties.halt_id
GROUP BY Haltestelle.name
ORDER BY COUNT(Properties.p_key) DESC
LIMIT 3;

f)

Geben Sie alle Fahrtmöglichkeiten (Haltestellenkombinationen) aus, bei denen sich beide Haltestellen in Zone 1 befinden.

xxxxxxxxxx
SELECT H1.name AS Start, H2.name AS Ziel
FROM Haltestelle AS H1
JOIN Haltestelle AS H2 ON H1.id != H2.id
WHERE H1.zone = 1 AND H2.zone = 1;

a)

Gegeben sei die Relation R(A,B,C,D,E,G,H,I) mit den folgenden Mengen an funktionalen Abhaengigkeiten:

F={A→BABCD→EEI→GEI→HACDI→EACDI→G}
i) Schlüsselkandidat
einfache Prüfung

  • {A,B,C,D}R+={ABCDE}, also kein Schlüssel(kandidat)

  • {A,C,D,G}R+={ACDGBE}, also kein Schlüssel(kandidat)

  • {A,C,D,I}R+={ACDIEGB}, also potentiell ein Schlüsselkandidat

    • noch zu zeigen das es minimal ist

  • {B,E,G,H}R+={BEGH}, also kein Schlüsselkandidat

genaue Prüfung

k = 1: {A}∗={A,B}

k = 2: {EI}∗={E,I,G,H}

k = 3: {AEI}∗={A,B,E,I,G,H}, {ACD}∗={A,C,D,B,E}

Da wir bis zur größe k=3 keinen Schlüsselkandidaten finden konnten und die Attributhülle von {A,C,D,I} alle Attribute umfasst, handelt es sich um einen Schlüsselkandidaten.

ii) 3.-Normalform durch Synthesealgorithmus

Das Ergbnis des Synthesealgorithmus ist:

(1)Z={(ACDIEGB,{ACDI}),(AB,{A}),(EIGH,{EI}),(ACDE,{ACD})}
